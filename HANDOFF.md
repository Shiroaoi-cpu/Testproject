# 引き継ぎ書（別端末・別セッション用）

このリポジトリで開発中の放置系ゲーム「木こりの森」の**現状と次にやること**をまとめたもの。
新しいセッション（iPad の Claude Code on the web など）で作業を再開するときは、まずこのファイルと `DESIGN.md` を読むこと。

---

## 1. プロジェクト概要

スマホ広告でよくある「木を切る→加工→販売→人を雇う→自動化→効率化→拡張」型の放置ゲームを、
ちゃんと遊べるゲームとして作っている。参考イメージはホワイトアウトサバイバルの広告ゲーム。

- 公開先（GitHub Pages）
  - ボタン版：https://shiroaoi-cpu.github.io/Testproject/
  - **動く版ステージ1：https://shiroaoi-cpu.github.io/Testproject/stage1.html**
- Android では PWA としてホーム画面にインストールして遊んでいる。

## 3. stage1.html の現在の仕様

**ループ**：木を切る → 製材する → 売る → お金 → 自動化を買う

- **序盤は3工程すべて手動**（ボタン：木を切る／製材する／売る）
- **自動化は購入で段階解禁**（すべて Lv0 スタート）
  - `木こり`＝自動伐採、`製材所`＝自動製材、`行商人`＝自動販売、`鋭い斧`＝手動伐採量+1
- **レート**：木こり 1.0 原木/秒/人、製材所 0.25 板材/秒/Lv（原木2消費）、行商人 0.3 板材/秒/Lv（板材1=10G）
  - 木こり1人が製材所1つの消費を上回る設計。原木が溜まったら製材所を増やす、という遊び
- **演出**
  - 主人公が斧を振る／丸太が飛ぶ／作業員が木↔製材所を往復
  - 製材所：**回転のこぎり**（Lv1以上で出現、`r = 6 + min(Lv,7)*2.3` でレベルに応じて大型化）
  - 販売所：**店員**（自動化前は主人公＝オレンジが販売時に一瞬登場／自動化後はモブ＝緑が常駐）
    レベルで**帽子が変化**（1:キャップ 2:ニット帽 3:とんがり帽 4:シルクハット 5+:王冠）
- **セーブ**：localStorage キー `kikori_stage1_v1`。**オフライン進行は2/3速度・最大8時間**
- **設計上の注意**：経済処理は `setInterval(logicTick,200)` で回し、`requestAnimationFrame` は描画専用。
  （アニメが止まる環境でも経済が進むようにするため。ここを混ぜないこと）

## 4. index.html（ボタン版）の要点

- 4系統：材木 → 釣り(Lv10) → 鉱石(Lv30) → 料理(Lv50)、共通XPで解禁
- **XPは「販売で得たGと同量」のみ**（生産・加工では入らない）。カーブ `40 * Lv^2.2`
- 炭の「火力」システム（オガ炭1／白炭3／備長炭5）、自動施設に稼働/停止トグル
- 解放時に OK! ボタン付きモーダル通知（複数解放はキュー表示）
- セーブキー `kikori_save_v2`

## 5. 開発ルール

- **デプロイのたびに `service-worker.js` の `CACHE = 'kikori-mori-vN'` の N を上げる**
  （上げないと、インストール済みPWAが古いキャッシュを表示し続ける）
- コミットは細かく、プッシュはユーザーの指示でまとめて行う運用
- 動作確認は実機ブラウザ推奨（プレビュー環境は rAF と setInterval が間引かれて遅く見える）
